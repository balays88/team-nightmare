# Workflow Specification: Pre-Mortem Analysis

**Module:** security
**Status:** DEFINED
**Created:** 2026-01-31

---

## Workflow Overview

**Goal:** Identify fatal flaws in the design before a single line of code is written.

**Description:** A "Reverse Brainstorming" session where we assume the project has already failed and determine the causes.

**Workflow Type:** Analysis / Brainstorming

---

## Planned Steps

| Step | Name       | Goal                                          |
| ---- | ---------- | --------------------------------------------- |
| 1    | The Wake   | Set the scene of failure (6 months in future) |
| 2    | The Ghosts | Brainstorm causes of death (Security, Logic)  |
| 3    | The Eulogy | Formal report of vulnerabilities              |
